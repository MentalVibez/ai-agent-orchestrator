Portfolio Demo

AI Agent Orchestrator

A production-focused multi-agent platform for diagnostics, automation, and governed AI operations. This demo shows what users see, what operators control, and what reliability guarantees are enforced.

Core Value

Turns plain-language goals into structured, auditable runs with policy controls and fail-safe behavior.

Who Uses It

Ops teams, IT engineers, and platform teams managing runbooks, incidents, and agent-driven diagnostics.

Proof Points

Branch protection, required checks, staged reliability gate, backup/restore gate, and explicit cost controls.

Live User Flow (3 Minutes)

1
Create a run: user posts a goal to /api/v1/run with an API key.
2
Planner orchestration: system chooses agent actions and tool calls with safety checks.
3
Trace + events: run status, events, and errors stream to persistent records for auditing.
4
Operational safeguards: budget caps, auth/rbac, idempotency, and retry/circuit-breakers enforce control.
5
Outcome: run completes/fails/cancels with explicit reason and recoverable state.

Architecture Snapshot

API Layer

FastAPI routes with auth, validation, and run lifecycle endpoints.

Planner + Agents

Goal-to-action orchestration with specialized agent profiles and tool routing.

State + Events

Run store, event log, queue/worker execution, and traceability for every decision.

Reliability Controls

Migrations checks, reliability gate drills, backup/restore gate, and protected main branch.

Demo Script (Terminal)

# 1) Health
curl -H "X-API-Key: $API_KEY" http://localhost:8000/api/v1/health

# 2) Start a run
curl -X POST http://localhost:8000/api/v1/run \
  -H "X-API-Key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"goal":"Diagnose high memory usage on worker node","agent_profile_id":"default"}'

# 3) Poll run status
curl -H "X-API-Key: $API_KEY" http://localhost:8000/api/v1/runs/<run_id>

# 4) View run history
curl -H "X-API-Key: $API_KEY" http://localhost:8000/api/v1/runs
